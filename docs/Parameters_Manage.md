# 对于参数设置的详细说明
首先是一些固定值，根据设备设置Batchsize为64，然后原始设置是并行得到训练数据和训练，1000步会测试一次，我们单一设备则是串行设计，首先会生成数据，然后训练1000步，然后比较。
首先定下的是一局中要进行多少局。由于资源紧张，我们设置样本占比batchsize=64,r=0.25新样本占比，一局满打满算100步数据，根据64*1000*r/100=160局。所以numEps=160，步数设为200，应该是比较合理的。
然后让新样本占比1/9设置池子容量，但其实是有一次其实是有bug的训练中开始想让池子容量设为100000个位置，然后一局满打满算100个位置，得到1000局的容量，大概是7次循环，但是当时六次后仍然没有超过1500分，所以就增加，后来改成小于1500则不排除老师数据。8种数据增强数据相关性高所以没有考虑。
训练之后大概一轮在2h所有，两天能得到24左右，再多给一点凑成30轮为总轮数，也据此做了学习率的调度。评估的两个参数都是为了在不会花费比获取数据更多的时间，即比真正为进化做努力的时间更长而选择，同时要是偶数。
平局的值是因为原项目没有考虑到平局，所有规则也没有考虑平局，为了安稳考虑，我选择了一个极小值。
ab_data设为1是因为这个耗时较长，一轮要4小时左右，所以仅做一轮。
